# 维修报价单系统 — 部署使用指南

## 📁 文件说明

```
repair-quote/
├── admin.html    ← 管理员：填写报价、生成链接
└── confirm.html  ← 客户：查看报价、签字确认
```

---

## 🚀 部署方式（3选1）

### 方式1：GitHub Pages（免费，推荐）

1. 去 GitHub 新建一个仓库，比如叫 `repair-quote`
2. 把 `admin.html` 和 `confirm.html` 上传到仓库
3. 进入仓库 → Settings → Pages → Source 选 `main` 分支
4. 几分钟后就能通过 `https://你的用户名.github.io/repair-quote/admin.html` 访问

### 方式2：腾讯云COS静态托管（免费额度）

1. 注册腾讯云，开通COS
2. 创建存储桶，设为**公有读私有写**
3. 上传两个HTML文件
4. 开启静态网站托管，获得访问域名

### 方式3：直接本地打开测试

双击 `admin.html` 即可在浏览器中打开测试（无需部署）。

---

## 📱 使用流程

### 管理员操作

```
1. 手机/电脑打开 admin.html
2. 填写客户姓名、服务类型、问题描述
3. 填写费用明细（可增删行）
4. 点击「生成报价单链接」
5. 复制链接 → 打开微信 → 粘贴发给客户
```

### 客户操作

```
1. 微信里点击链接
2. 看到报价单详情（服务信息 + 费用明细）
3. 勾选同意协议 → 手写签字
4. 点击「确认签字」
5. 完成！
```

### 管理员查看签字

客户签字后，打开同一个链接即可看到签字结果。
（签字数据存在客户手机的浏览器本地，管理员查看需客户截图或转发确认截图）

---

## ⚠️ 注意事项

1. **数据存储**：当前版本数据通过URL传递+浏览器本地存储，适合小规模使用
2. **无服务号也能用**：链接需手动通过微信发给客户
3. **后续升级路径**：订单量大了之后，可以升级为服务号+后端服务器的完整方案
4. **安全提示**：链接包含报价信息，不要转发给无关人员
